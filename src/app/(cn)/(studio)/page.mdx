# ScienceOL Studio 文档

**ScienceOL Studio** 取自 **Science O**n**L**ine 之意，中文名 **科学在线工作室**。英文缩写 **SciOL**(/ˈsaɪ.ɔl/)。

我们致力于构建一个完全开源开放的通用 AI for Science 智能实验室系统，迈向科学通用智能 (Scientific General Intelligence, SGI) 未来。

我们的业务线由 **OL** 延伸为 2 部分，**O**pen**L**ab 与 **O**nline**L**ab。
**OpenLab** 致力于提升实验室自动化水平，以模块化硬件+开源系统加速智能改造升级。
**OnlineLab** 提供基于开源系统分布式的在线云实验室服务，兼顾科学计算与实验，并实现跨区域的实验室资源调配。
两者协同构建全方位的智能实验室生态系统。

我们的产品线包括：多环境管理与社区入口 Studio，AI 原生的容器化工作流调度系统 PROTIUM, 面向实验室场景的智能体 Xyzen，及 3D 数字孪生平台 Anti（即将推出）。实验室硬件基座使用开源的 [Uni-Lab-OS](https://github.com/dptech-corp/Uni-Lab-OS) 项目。所有产品均在 GitHub 开源。

本页面是关于**多环境管理与社区入口 Studio** 的帮助文档，了解更多模块的信息，请导航至其它主题。

## 快速开始 {{ anchor: false }}

开始使用 ScienceOL Studio，首先在 [仪表板 -> 工作流](#) 中创建一个工作流程，
然后跟随我们的 [工作流快速开始指南](#) 创建您的第一个演示工作流。 {{ className: 'lead' }} 
当你的工作流准备就绪后，可以将其发布到我们的[社区](#)，以帮助更多人发现它。 {{ className: 'lead' }}

运行工作流需要[计算环境](#)或[实验环境](#)，按照提示设置你的 API 密钥和环境配置。 {{ className: 'lead' }}

你可以在 GitHub Issue 中提出问题。此外，我们为提交了 PR 并通过审核的用户提供无限量的核心开发者答疑服务。遇到了问题？那就来提 PR 吧！
